Contents of login.config:

 

<?xml version="1.0"?>

<SagaLogin>

<host>127.0.0.1</host>

 

<!-- address of SQL/db4o server or Path to Save for dat implementation-->

<dbhost>127.0.0.1</dbhost>

 

<!-- database name for SQL -->

<dbname>sagalogin</dbname>

 

<!-- port for databaser server(SQL or db4o) -->

<dbport>8000</dbport>

 

<!--

Database mode,

0:db4o

1:MySQL

2:Dat

3:MSSQL

  -->

<ifSQL>2</ifSQL>

 

<dbuser>saga</dbuser>

<dbpass>saga</dbpass>

 

<!-- port for login server -->

<port>6000</port>

 

<!-- if no db4oserver then use this file to store inforamtions,db4o mode only -->

<userdbfile>char.db</userdbfile>

<chardbfile>char.db</chardbfile>

 

<mapserverpass>secret</mapserverpass>

</SagaLogin>

 

host: IP Address of login server

port: Port of login server

dbhost: IP Address for MySQL/db4o server

         or Path of saving for dat DB mode(example: ".\" or "c:\save\".)

dbname: Database name in MySQL database

dbport Port for MySQL/db4o server.  Only valid for MySQL or DB4O mode

 

ifSQL: this field is a little bit confusing, this is actually DB Mode, it has 3 options

        0: DB4O - not supported in current version of Saga

        1: MySQL

        2: Dat  - A file based saving solution, it's the simplest to use.

 

dbuser: User for DB server. Only valid for MySQL or DB4O mode

dbpass: Password for DB server.  Only valid for MySQL or DB4O mode

 

userdbfile: If there is no DB server set, use this file to save data.  Only valid for DB4O mode

chardbfile: If there is no DB server set, use this file to save data.  Only valid for DB4O mode

 

mapserverpass: Password for mapserver to login
